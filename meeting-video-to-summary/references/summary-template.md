# Meeting Ink 摘要模板

此為 Claude 產出 `summary.md` 時必須遵循的結構。所有內容都必須能從 `transcript.txt` 追溯；找不到依據的欄位請標註「待確認」而不要捏造。

---

## 使用方式

- 將「{{變數}}」替換為實際內容
- 若某個子區塊本次會議沒有內容，保留該 heading 並在下方寫「本次會議無此項」
- 表格若只有 0 筆資料，保留表頭即可
- 所有中文內容使用繁體中文

## 填寫要點

### 頁首 metadata
- **建立時間** = 今天日期（摘要產出日）
- **音檔時長** = pipeline 輸出的 `audio duration`
- **與會者** = 從逐字稿中辨識到的說話者 + 被點名的人；若使用者提供名單以使用者為準

### TODO 三個子分類的差別
- **原定要完成的部分**：上次會議就排定、本次會議仍在追蹤的工作
- **新增加的部分**：本次會議才新冒出來的工作
- **潛在要做但還沒規劃時間的部分**：有討論但未定 deadline、未明確指派的項目

若逐字稿內容分不出這三類，可先全部放入「新增加的部分」。

### DONE 區
每項採三段式：主題 → 狀態 → 延伸下一步。
- 「狀態」是本次會議對此項的結論（誰確認、確認了什麼）
- 「延伸下一步」是後續要串接的動作（而非 TODO 本身，TODO 應放在 TODO 區）

### 其他相關資訊
- 依實際議題數分多個 H3 heading，至少 2 個、通常 5~8 個
- 每個議題下先一段情境說明（2~4 句），再條列細節
- 細節條列多採「**標籤**：內容」的鍵值形式，讓讀者掃視即可抓重點
- 需要再細分時可用 H4 作為子議題，深度最多兩層

### 不可以做的事
- 不要揣測未在逐字稿中提到的人名、日期、數字
- 不要把討論中的「可能」、「考慮」寫成「確認」
- 不要合併多個不同議題的討論到同一段落
