# Finance Excel Review Skill

財務記帳 Excel 逐筆審核工作流程。依序完成每筆資料的三步驟審核。

## 必要工具

所有 Excel 操作**必須使用 excel-xlsx skill**。禁止以其他方式讀寫 Excel。

## Excel 位置

```
/media/che-an-wu/新增磁碟區4/giintaipei/myspace/收支/Family Finance_2025 - For_openclaw.xlsx
```

規則：直接修改原始檔案，不複製、不備份、不建立新檔案。

## 工作表範圍

只處理工作表 `1` ~ `12`（代表 1–12 月）。忽略其他工作表。

## 審核狀態判斷

- 「項次」欄位**空白** → 尚未審核
- 「項次」欄位**有值** → 已審核完成

只處理**項次為空白**的資料。

---

## 批次筆數設定

- **預設批次筆數：5 筆**
- 使用者可隨時說「每次審核 N 筆」來更改批次筆數，立即生效
- 批次筆數變更後回覆確認，例如「已將每次審核筆數改為 N 筆」

## 審核開始流程

使用者開始審核時，執行：

1. 計算進度：`已審核筆數 / 總筆數`，回傳百分比
2. 找出**最舊的 N 筆未審核資料**（N = 當前批次筆數），顯示欄位：項目、金額、分類、方式、日、備註
3. 若任一筆資料字體為紅色 → 提醒「第 X 筆資料為紅字標記」
4. 顯示這 N 筆資料，加上**第一筆的前 N 筆**及**最後一筆的後 N 筆**作為對照（最多共 3N 筆）的 ASCII plain text table

### ASCII 表格格式規範

```
+------+------+--------+------+------+-----+------+
| 項次 | 項目 | 金額   | 分類 | 方式 | 日  | 備註 |
+------+------+--------+------+------+-----+------+
|      |      |        |      |      |     |      |
+------+------+--------+------+------+-----+------+
```

規則：
1. 使用 `+`、`-`、`|` 組成框線
2. 包含欄位標題列
3. 每列資料前後皆有分隔線
4. 禁止使用 Markdown table
5. 不得省略欄位

---

## 三步驟審核工作流程

每個批次的 N 筆資料，依序逐筆完成三步驟，**每筆每步驟都必須等待使用者確認後再繼續**。

### Step 1 — 保留或刪除（批次顯示，逐筆確認）

先列出本批次所有 N 筆資料，詢問：「請確認每筆是否保留（可用「全部保留」或逐筆指定，例如「第2筆刪除」）」

| 使用者選擇 | 操作 |
|---|---|
| 保留 | 將該列所有 cell 背景色設為 `#FDEADA`（不動其他列） |
| 不保留 | 刪除該筆資料列 |

### Step 2 — 資料內容確認（批次）

列出本批次所有保留的資料，詢問：「以上資料內容是否皆正確？可指定修改（例如「第2筆金額改為500」）或標記不確定（例如「第3筆不確定」）」

| 情況 | 操作 |
|---|---|
| 全部正確 | 整批繼續 Step 3 |
| 需要修改 | 依使用者指示修改對應筆的欄位（項次/項目/金額/分類/方式/日/備註），修改完成後繼續 Step 3 |
| 標記不確定 | 將該筆所有欄位字體改為**紅色**（原本已是紅色則保持不變），該筆**跳過 Step 3** |

### Step 3 — 固定收支判斷（批次）

列出本批次通過 Step 2 的資料，詢問：「以上哪些筆為固定收入或固定支出？請指定（例如「第1筆固定支出」），無則回覆「無」」

| 情況 | 操作 |
|---|---|
| 指定固定收入 | 將該筆項目/金額/日填入工作表上方「固定收支」區塊 |
| 指定固定支出 | 將該筆項目/金額/日填入工作表上方「固定支出」區塊 |
| 無 | 不額外操作 |

Step 3 完成後，填寫本批次所有已審核資料的「項次」欄位（依該列在工作表中的相對位置順序填寫），代表審核完成。

---

## 完成規則

一個批次全部完成後：

- 若仍有未審核資料 → 自動顯示下一批次並重複流程
- 若全部審核完畢 → 回覆「該月份所有資料已審核完成」

---

## 行為準則

1. 每次處理一個批次（N 筆），批次內逐筆進行三步驟
2. 每個步驟都必須等待使用者確認
3. 不自行猜測資料內容
4. Excel 操作必須透過 excel-xlsx skill
5. 不修改未被審核的其他資料列
