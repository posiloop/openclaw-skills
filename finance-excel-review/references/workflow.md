# Finance Excel Review — Workflow Reference

## 工作表欄位對應

每月工作表（1~12）的資料列欄位（依序）：

| 欄位 | 說明 |
|---|---|
| 項次 | 審核序號，空白代表未審核 |
| 項目 | 消費項目名稱 |
| 金額 | 金額數字 |
| 分類 | 費用分類 |
| 方式 | 付款方式 |
| 日 | 日期（日） |
| 備註 | 補充備註 |

## 工作表上方固定收支區塊

每月工作表上方設有兩個固定區塊：

- **固定收支**（固定收入）：填入固定收入項目/金額/日
- **固定支出**：填入固定支出項目/金額/日

當 Step 3 判斷為固定收入或支出時，將資料填入對應區塊。

## 「項次」填寫規則

項次代表該筆資料在當月的審核序號。
填寫方式：依照該資料列在工作表中的**相對位置順序**填入數字（如第 1 筆未審核資料填 1，依序遞增）。

```

| 項次 | 項目     | 金額   | 分類   | 方式   | 日  | 備註     |

|  1   | 早餐     |   80   | 餐飲   | 現金   |  3  |          |
+------+----------+--------+--------+--------+-----+----------+
```

## 紅字資料處理

若最舊未審核資料的字體為紅色（由 excel-xlsx skill 讀取 font color 屬性判斷），必須在回傳資訊時加上提醒：

「⚠️ 此筆資料為紅字標記」

Step 2 使用者選擇「不確定」時：
- 若字體已是紅色 → 保持原樣
- 若字體不是紅色 → 將所有欄位字體改為紅色

## 狀態流程圖

```
開始審核
  │
  ├─ 計算進度（已審核/總筆數 %）
  └─ 找最舊未審核資料
       │
       ├─ [Step 1] 保留？
       │     ├─ 是 → 設背景色 #FDEADA → [Step 2]
       │     └─ 否 → 刪除資料列 → 下一筆
       │
       ├─ [Step 2] 內容正確？
       │     ├─ 是 → [Step 3]
       │     ├─ 修改 → 依指示修改 → [Step 3]
       │     └─ 不確定 → 設字體紅色 → 直接下一筆（跳過 Step 3）
       │
       └─ [Step 3] 固定收支？
             ├─ 固定收入 → 填入固定收支區塊 → 填項次 → 下一筆
             ├─ 固定支出 → 填入固定支出區塊 → 填項次 → 下一筆
             └─ 否 → 填項次 → 下一筆
```
