# 報帳 Skill

## Session 記憶共用規範

適用於本 skill 內所有多步驟欄位確認與流程分流。

- 同一個 session 內，凡是使用者已明確確認過、且合理可沿用的流程性資訊，後續相同類型任務應直接沿用，不再重複詢問。
- 若使用者在同一個 session 中主動更改先前設定，必須立即以最新答案覆蓋舊值，後續沿用新值。
- 僅記住「流程選擇」與「可穩定沿用的設定值」，不要記住每一筆單據各自應重新確認的內容。
- 若新上傳的資料與前文脈絡明顯屬於同一批次，預設沿用既有流程；若看起來已切換到新任務、跨批次、或使用者語意不明，再補問一次。
- 若沿用 session 記憶可能造成寫入錯誤，優先詢問確認，不可硬套。

**本 skill 必須記住的 session 資訊：**
- 報帳流程是否已確認為「一般報帳」
- 報帳公司（實見 / 益循）

**本 skill 不可沿用的資訊：**
- 報帳姓名。每一筆報帳都必須重新詢問，不得沿用前一筆答案。

## 公司選擇（第一步）

報帳流程開始時，需確認要報帳到哪間公司：
- **實見** → 寫入 Google Sheet「實見_2026報帳_openclaw」
- **益循** → 寫入 Google Sheet「益循生活_2026報帳_openclaw」

確認公司後，才進入後續流程。

## 兩種報帳方式

### 方式一：上傳圖片

1. 使用者上傳發票、銷貨單、稅額繳款書或其他可作為支出憑證的單據圖片
2. 若本次對話 session 已經確認本批單據走「一般報帳」流程，後續相同 session 內的發票/收據/稅額繳款書圖片直接沿用，不再重複詢問。
   若本次對話 session 已經確認改走「小龍蝦零用金發票登錄」流程，則停止本 skill，改交由 `lobster-invoice-intake` 處理。
   僅在 session 尚未確認流程，且使用者僅上傳單據圖片，或訊息不足以判斷是「一般報帳」還是「小龍蝦零用金發票登錄」流程時，才先詢問：
   「這張單據要走哪個流程？
   1. 一般報帳（實見/益循）
   2. 小龍蝦零用金發票登錄」
   - 若使用者選擇 2，停止本 skill，改交由 `lobster-invoice-intake` 處理
   - 若使用者選擇 1，記錄本 session 之後續單據沿用一般報帳流程，不再重複詢問
   - 僅在使用者選擇 1，或上下文已明確表示是一般報帳流程時，才繼續以下步驟
3. 只要圖片屬於一般報帳流程，且是發票、銷貨單、稅額繳款書、扣繳稅額繳款書、其他政府規費或稅費繳納憑證等可作為支出證明的單據，應先直接使用 image 工具從圖片整理可辨識欄位，不可一開始要求使用者手動提供公司名稱、項目、金額、日期等基本欄位。
4. 若使用者已上傳圖片，即使後續只簡短回覆「1」選擇一般報帳流程，也應直接延續圖片報帳分支並立刻進行圖片辨識，不可因為使用者尚未手動補充欄位就切換成手動輸入流程。
5. 對於稅額繳款書、扣繳稅額繳款書、保費繳款單、退休金繳款單、其他政府規費或稅費繳納憑證，預設一律先自動從圖片整理欄位，再請使用者確認；這是固定流程，不應先要求使用者手動輸入基本欄位。
6. 使用 image 工具分析圖片，先萃取以下欄位：
   - 單據日期（圖片識別格式：MMDD，用於圖片檔名；寫入 Sheet 格式：MM/DD/YYYY）
   - 公司名稱（一般單據優先取付款、開立、扣繳或賣方單位；若為稅額繳款書，固定取最上方機關名稱）
   - 項目（費用類別，如電費、餐費、交通費、扣繳稅款）
   - 金額（數字，不含幣別符號）
7. 若單據為稅額繳款書、保費繳款單、退休金繳款單、其他政府規費或稅費繳納憑證，視為可報帳憑證，並依下列預設規則整理：
   - 日期：一律優先以蓋章區塊、收款章、代收章或收據聯上的實際收款日期為準；若該日期無法可靠辨識，再回頭請使用者確認，不可直接改抓給付日期、列印日期、繳款期限或其他日期代替
   - 公司名稱：優先取票面最上方或主要收款機關名稱；若為稅額繳款書，固定取最上方機關名稱，例如「財政部臺北國稅局」；不要改抓扣繳單位名稱
   - 項目：預設寫成可讀費用名稱，例如「扣繳稅款」；若票面可明確辨識費用種類，可整理成「扣繳稅款（執行業務報酬）」、「勞工退休金雇主提繳」、「全民健康保險費補繳」等
   - 金額：優先取實際繳納或應繳總金額
   - 若缺少必要欄位，不可臆測，需回頭向使用者確認
8. 將圖片整理結果先回傳給使用者，詢問「以上資訊是否正確？」。在這一步之前，不可先要求使用者手動提供報帳公司、報帳姓名、開銷單位、無發票原因等欄位。
9. 若使用者確認正確，或完成修正後，再補問寫入所需但無法穩定由圖片直接判定的欄位，順序如下：
   - **確認報帳公司**：若 session 中已有記錄則直接沿用（不再詢問），否則詢問使用者（實見 or 益循）
   - 若使用者選擇**益循**，接著詢問「請問開銷單位是？」，**必須等使用者回答後才能進入下一步**
   - 詢問「請問報帳姓名是？」
10. 使用者提供完成寫入所需欄位後，執行以下動作（同步進行）：
   a. 將資料寫入對應公司的 Google Sheet 對應月份工作表；一般發票、銷貨單等單據的「有無發票」填「是」，但若單據為稅額繳款書、扣繳稅額繳款書、政府規費或稅費繳納憑證，則「有無發票」填「否」；「無發票原因」留空；資料填入工作表最上方的第一個可用列，不依流水號排序；嚴禁自行新增欄位，只填寫工作表中既有的欄位
   b. 從該月份工作表的 B1 儲存格讀取 Google Drive 資料夾連結，從連結中萃取資料夾 ID，將圖片上傳至該資料夾；檔名格式為 `MMDD $金額_項目.jpg`（如 `0218 $2099_電費.jpg`）
11. 完成後告知使用者已完成報帳

### 方式二：手動輸入

1. 使用者直接提供：發票日期、姓名、公司名稱、項目、金額
2. **確認報帳公司**：若 session 中已有記錄則直接沿用（不再詢問），否則詢問使用者（實見 or 益循）
3. 若使用者選擇**益循**，接著詢問「請問開銷單位是？」，**必須等使用者回答後才能進入下一步**
4. 將資料寫入對應公司的 Google Sheet 對應月份工作表，「有無發票」填「否」；資料填入工作表最上方的第一個可用列，不依流水號排序；嚴禁自行新增欄位，只填寫工作表中既有的欄位
5. 詢問使用者：「請問無發票的原因是？」
6. 將使用者回覆填入「無發票原因」欄位
7. 完成後告知使用者已完成報帳

## 工具使用

- 分析圖片：使用 `image` 工具
- 讀寫 Google Sheets：使用 `gog sheets` CLI
- 上傳 Google Drive：使用 `gog drive upload` CLI

## 寫入列位置判斷（重要）

工作表 A 欄已預填流水號（每日一列），B 欄（發票日期）才是實際資料欄位。
寫入時，必須透過以下步驟找到正確列：

1. 讀取工作表資料欄範圍：
   - 實見：`gog sheets get <sheetId> "YYYY-MM!B:H" --json`
   - 益循：`gog sheets get <sheetId> "YYYY-MM!B:I" --json`
2. 從第 3 列起逐列掃描，找出第一個「主要資料欄位皆為空白」的列號。
3. 判斷規則如下：
   - 實見以 B 到 F 欄（發票日期、姓名、公司名稱、項目、金額）是否皆為空白作為可用列判斷
   - 益循以 B 到 G 欄（發票日期、姓名、公司名稱、開銷單位、項目、金額）是否皆為空白作為可用列判斷
   - **不可使用整列全空判斷**，因為 G/H/I 等後段欄位可能有預設值、公式、資料驗證或既存內容
   - 即使「有無發票」或「無發票原因」欄位已有預填值，只要主要資料欄位為空，仍視為可用列
4. 將資料從該列的 B 欄開始寫入（依據該公司的欄位順序）

嚴禁依照流水號或日期推算列號，也嚴禁用整列全空作為判斷；必須以主要資料欄位是否為空白為準。

## Drive 資料夾動態讀取（重要）

每次上傳圖片時，**不要使用寫死的資料夾 ID**，而是動態從工作表讀取：

1. 讀取該月份工作表的 B1 儲存格：`gog sheets get <sheetId> "YYYY-MM!B1" --json`
2. B1 的值為 Google Drive 資料夾連結，格式如：`https://drive.google.com/drive/folders/<folderId>?usp=drive_link`
3. 從連結中萃取資料夾 ID（`/folders/` 後、`?` 前的部分）
4. 使用該資料夾 ID 作為上傳目標：`gog drive upload <圖片路徑> --name "MMDD $金額_項目.jpg" --parent <folderId> --json`

## 注意事項

- 圖片報帳流程的第一個動作是先分析圖片並整理日期、公司名稱、項目、金額，再向使用者確認；不可在這之前先要求使用者手動補公司、姓名、開銷單位、無發票原因等欄位，除非影像真的無法可靠辨識且該欄位又屬於圖片可判讀的基本欄位
- 若使用者先上傳圖片，再以數字選項確認走一般報帳流程，仍視為圖片報帳流程，必須直接接續做圖片辨識，不可誤切成手動輸入流程
- 只有在圖片整理結果經使用者確認後，才能再補問報帳公司、開銷單位、報帳姓名等寫入所需欄位，並據此決定寫入哪個 Sheet
- 分析圖片後，必須等使用者確認或修正，才能寫入 Sheet 及上傳圖片
- 一般報帳流程下，凡使用者上傳可作為支出證明的單據圖片時，必須先嘗試從圖片自動整理日期、公司名稱、項目、金額，再向使用者確認；不可先要求使用者手動補這些基本欄位，除非影像真的無法可靠辨識
- 對於稅額繳款書、扣繳稅額繳款書、保費繳款單、退休金繳款單、其他政府規費或稅費繳納憑證，預設流程就是先從圖片自動整理欄位，再請使用者確認
- 稅額繳款書、扣繳稅額繳款書、保費繳款單、退休金繳款單、其他政府規費或稅費繳納憑證，只要可作為支出證明，皆可走一般報帳流程
- 稅額繳款書、保費繳款單、退休金繳款單等政府或保費類單據，日期一律優先抓票面蓋章區塊、收款章、代收章或收據聯上的實際收款日期；若該日期不清楚，必須向使用者確認，不得自行改用列印日期、繳款期限或其他日期
- 稅額繳款書的公司名稱一律抓票面最上方的機關名稱，例如「財政部臺北國稅局」；其他政府或保費類單據則優先抓主要收款機關名稱，不要誤抓投保單位、扣繳單位、申報單位或其他下方公司資料
- 若單據不是傳統發票，仍可依單據性質作為正式支出憑證處理；但稅額繳款書、扣繳稅額繳款書、政府規費、保費或退休金繳納憑證的「有無發票」一律填「否」
- 月份工作表不存在時，應提示使用者或嘗試建立
- 金額欄位僅填數字，不含貨幣符號
- 日期欄位寫入 Sheet 時格式為 MM/DD/YYYY（如 02/18/2026）
- 圖片檔名中的日期格式為 MMDD（如 0218）
- 所有 Google Sheet 操作一律透過 Google Sheets API 直接修改線上檔案，嚴禁下載至本地端後再修改
- 益循比實見多一個「開銷單位」欄位，寫入時注意欄位位移
- 找寫入列時，不可用整列全空判斷，必須只看主要資料欄位是否為空；因為「有無發票」或其他後段欄位可能已預填預設值

## 已知 Bug：gog sheets update 多值串接問題

**問題：** `gog sheets update <sheetId> <range> val1 val2 val3 ...` 這種多值寫法，會將所有值串接後全部寫入 range 的第一格（如 B3），而非依序分佈至 B3、C3、D3 等欄位。

**錯誤範例（禁用）：**
```bash
gog sheets update <sheetId> "2026-03!B3:H3" "03/19/2026" "姜智森" "中華郵政" "掛號郵件" "160" "是" ""
# 結果：B3 = "03/19/2026姜智森中華郵政掛號郵件160是"，C3~H3 為空
```

**正確做法：每格分開寫入**
```bash
gog sheets update <sheetId> "2026-03!B3" "03/19/2026" --json
gog sheets update <sheetId> "2026-03!C3" "姜智森" --json
gog sheets update <sheetId> "2026-03!D3" "中華郵政" --json
gog sheets update <sheetId> "2026-03!E3" "掛號郵件" --json
gog sheets update <sheetId> "2026-03!F3" "160" --json
gog sheets update <sheetId> "2026-03!G3" "是" --json
```

每個欄位必須單獨呼叫一次 `gog sheets update`，不可合併為一次多值指令。
