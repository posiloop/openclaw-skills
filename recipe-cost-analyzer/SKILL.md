## 食譜 Schema（建全 2026-04-27 規格）

每個食材一個欄目，最後合計：

| 欄位 | 說明 | 預設 / 公式 |
|---|---|---|
| 食譜名稱 | 例：「番茄燉牛肉」 | — |
| 站點 | 中科15B / 鴻海 / 台北央廚 | — |
| 內容物 | 食材名稱（取原料名稱） | — |
| 供應商 | 從 uMeal 價格表帶 | — |
| 用量 | 原料用量 | — |
| 單位 | KG / L / 顆 / 片 等 | **預設 KG 或 L** |
| 步留率 | 食材步留率 | **若未告知 → 默認 0.98**（價格表也有步留率，可作為參考） |
| 單價 | 從 uMeal `最小單位價格` 帶 | 自動 |
| 成本 | 單筆食材成本 | **內容物單價 × 用量 × 步留率** |

合計列：

| 欄位 | 公式 |
|---|---|
| 總成本 | Σ(每筆 成本) |
| 總投入量 | Σ(每筆 用量；以 KG 換算) |
| 製成率 | **若未告知 → 默認 0.90** |
| 總製成量 | 總投入量 × 製成率 |
| 單位成本 | 總成本 / 總製成量 |
| 製作流程說明 | 自由文字 |

## 餐盒 Schema（建全 2026-04-27 規格）

| 欄位 | 說明 | 預設 / 公式 |
|---|---|---|
| 餐盒名稱 | 例：「番茄燉牛肉便當」 | — |
| 站點 | 中科15B / 鴻海 等 | — |
| 內容物 | 取**食譜生產物**（已建食譜的單位成本）或原料名稱 | — |
| 用量 | 該內容物用量 | — |
| 單位 | KG / L / 顆 等 | **預設 KG 或 L** |
| 步留率 | 內容物步留率 | **若未告知 → 默認 1.00**（餐盒組裝時通常無耗損） |
| 單價（成本） | 取對應食譜的**單位成本**或原料單價 | 自動 |
| 成本 | 單筆內容物成本 | 內容物單價 × 用量 × 步留率 |

合計列：

| 欄位 | 公式 |
|---|---|
| 總成本 | Σ(每筆 成本) |
| 售價 | 餐盒售價（含稅） | — |
| 未稅價 | 售價 / 1.05 |
| 原價率% | 總成本 / 未稅價 × 100% |

## 工作流程

### 1. 新增食譜
1. 收集：食譜名稱、站點、食材清單（每項：品名、用量、單位、步留率（可選））、製成率（可選）、總投入量（可選）、製作流程
2. 對每項食材：
   - 在 uMeal 價格表（先找本地 cache，無則重新下載）跨 8 個 sheet 用 fuzzy match 找到最相近的「輸入名稱」
   - 取出供應商、最小單位價格、單位
   - 步留率：**recipe 自身指定 > uMeal 價格表 > 預設 0.98**
   - 計算 cost = 單價 × 用量 × 步留率
3. 計算總成本 / 總製成量 / 單位成本
4. 寫入 JSON：`~/work/yixun-sheng-huo/recipes/<站點>/<recipe_id>.json`
5. 同步 Google Sheet：append 到 `食譜明細` 與 `食譜彙總` 分頁
6. 產 PDF 成本表，回傳

### 2. 新增餐盒
1. 收集：餐盒名稱、站點、內容物清單、售價
2. 對每項內容物：
   - 若內容物是「食譜生產物」→ 從本地 JSON 讀取對應食譜的 `unit_cost_per_kg`
   - 若是原料 → 直接從 uMeal 帶單價
   - 步留率預設 1.00（除非餐盒有指定）
3. 計算總成本、未稅價（售價/1.05）、原價率（總成本/未稅價×100%）
4. 寫入 JSON：`~/work/yixun-sheng-huo/recipes/<站點>/bento_<bento_id>.json`
5. 同步 Google Sheet：append 到 `餐盒明細` 與 `餐盒彙總` 分頁
6. 產 PDF 成本表，回傳

### 3. 修改 / 重算
- 食譜或餐盒被修改時，自動重算所有引用該食譜的餐盒（連動）
- 價格表更新時可重算所有食譜（重新下載 cache → 對所有 JSON 重跑）

### 4. 查詢
- 列出某站點所有食譜或餐盒
- 列出原價率 > X% 的餐盒（標註「成本過高」）
- 列出引用某食譜的餐盒

## 預設 / 規則

- 站點分類**必填**；若使用者沒提，預設為「中科15B」並向使用者確認
- 單位預設 KG / L
- 食譜步留率預設 0.98；餐盒步留率預設 1.00
- 製成率預設 0.90
- 售價輸入時假設**含稅**；公司端要報的是未稅，因此自動算 售價 / 1.05
- 找不到價格時必問建全，不要自己估
- PDF 採標楷體（沿用「報表 PDF 為主」規則）
- Google Sheet 同步時，不要動使用者既有列；新增的 row 加在 sheet 最下方下一可用列（仿照中科叫貨表的「先讀完整、再寫入」規則）

## 缺欄處理

當下列欄位無法可靠取得時，**先詢問建全再寫檔**：
- 食譜名稱、站點
- 食材對不上 uMeal 的品項（提供 2–3 個最相近候選讓建全選）
- 步留率特殊（例如帶骨肉處理損耗大）
- 售價未明確
- 製作流程很複雜想簡化敘述時

## 擴充

- 新站點：在「適用範圍」表格新增即可
- 新供應商：uMeal 價格表自然涵蓋；不需動 skill
- 新欄位：在 SKILL.md 與 JSON schema 同步加，並通知 Google Sheet 表頭調整
- 跨公司（例：實見）：未來建獨立 skill 或在此 skill 加 `company` 維度
