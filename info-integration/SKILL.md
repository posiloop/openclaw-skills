# Info Integration Skill

五階段完整工作流程，從 Google Drive 素材到最終產出。

## 流程概覽

**Phase 0** → 素材清單摘要  
**Phase 1** → 確認主軸與執行策略  
**Phase 2** → Prototype + 分階段計畫  
**Phase 3-N** → 逐階段執行  
**Final** → 成品 + 流程優化建議

---

## Phase 0：素材盤點

收到 Google Drive 連結後：

1. 用 `web_fetch` 或 `browser` 列出資料夾內所有檔案（含子資料夾）
2. 對每個檔案產出一條簡短摘要（1-3 句）：
   - **影片/音訊**：主題、時長（若可得）、重點內容
   - **文字/PDF**：主題、頁數/字數（若可得）、核心論點
   - **圖片**：拍攝主題或圖表類型、視覺重點
3. 以表格或清單呈現：`檔名 | 類型 | 摘要`
4. 請用戶確認：「這些摘要有沒有抓到重點？有需要補充或刪減的素材嗎？」

> 無法直接存取的檔案（如受權限保護），列出並提示用戶調整共用設定或手動提供內容。

---

## Phase 1：主軸與策略確認

完成素材確認後：

### 1a. 提出主軸建議

依素材性質提出 2-4 個可能主軸，例如：
- 資訊整合報告（適合文字/數據為主的素材）
- 教學影片或簡報（適合有步驟性知識的素材）
- 回憶剪輯 / 紀念影片（適合照片/影片日記素材）
- 品牌/行銷提案（適合產品或服務素材）

### 1b. 提出呈現方式建議

- 文字報告（Markdown / PDF）
- Google Slides / PowerPoint 簡報
- 影片（剪輯、字幕、配音）
- 互動網頁 / Canvas 展示

### 1c. 分析可行性與工具需求

列出每個組合需要的工具和環境，例如：
- Google Slides：可用 `browser` 直接操作或產生 Markdown 轉換
- 影片剪輯：需要 ffmpeg、MoviePy（建議安裝）或串接外部服務
- PDF 摘要：已有 `pdf` 工具可直接使用
- 圖片處理：已有 `image` 工具
- 語音生成：已有 `tts` 工具

### 1d. 列出至少 3 個完整提案

每個提案包含：主軸 + 呈現方式 + 所需工具 + 預估工作量（簡/中/複雜）

請用戶選擇或調整提案後再進入 Phase 2。

---

## Phase 2：Prototype + 分階段計畫

確認執行策略後：

1. **立即產出 Prototype（第一版雛形）**
   - 內容精簡但結構完整，足以讓用戶感受最終成品樣貌
   - 例如：簡報的前3頁、報告的骨架、影片的第一個片段腳本

2. **提出 2-5 個後續執行階段**
   - 每個階段有清楚目標和預期產出
   - 例如：「第一階段：完成所有章節文字草稿」「第二階段：加入圖表與視覺化」

3. 請用戶確認：「這個 prototype 方向對嗎？是否繼續執行這個計畫，或換其他提案？」

---

## Phase 3-N：逐階段執行

每個階段：

1. 執行該階段的完整工作
2. 呈現成果並說明做了什麼
3. 詢問：「這個階段有什麼需要調整或修改的嗎？」
4. 根據反饋調整後進入下一階段

---

## Final：成品交付 + 流程優化

1. 交付最終成品（檔案、連結、或文字）
2. 回顧整個流程，基於本次素材特性和執行經驗，提出：
   - 下次同類型主軸/呈現方式的流程改進建議
   - 可預先準備的素材格式或標準化需求
3. 詢問用戶是否同意將優化建議更新至此 Skill
4. 若同意，更新 `SKILL.md` 中的相關步驟或新增 `references/` 筆記

---

## 注意事項

- **每個 Phase 結束都要等用戶確認再繼續**，不自動跳到下一階段
- Google Drive 存取：優先用 `web_fetch`；若需登入則用 `browser`（profile="user"）
- 大型影片檔案無法直接分析時，請用戶提供字幕檔（.srt/.vtt）或文字稿
- 優先使用已有的工具（`pdf`、`image`、`tts`、`browser`），對需要額外安裝的工具明確告知用戶
- 語言：全程使用繁體中文回應（除非用戶指定其他語言）
